* Records
:PROPERTIES:
:ORDER: 30
:KIND: records
:SCHEMA: %ITEM(Task) %TODO(Status) %PRIORITY %DEADLINE %SCHEDULED %TAGS %EFFORT %CATEGORY %Assignee(Who) %Room
:COLTYPES: text enum(TODO,NEXT,DONE,CANCELLED) enum(A,B,C) date date text text text text enum(Kitchen,Garage,Garden)
:FILTER: (not (done))
:ACTIONS: todo(DONE) schedule deadline tags(home,errand) priority(A) refile(* Archive) archive(subtree)
:GROUP: Tasks
:END:

Direct children are the records: all eight schema specials, custom
drawer properties, a filter, and the full action vocabulary.

:SCHEMA: %ITEM(Title) %CATEGORY(Category)
:COLTYPES: text text
:NAV: drawer

A vulpea vault directory: one .org note file per record.  Without
vulpea installed the view degrades to a placeholder — the app still
loads on a bare jetpacs-core.  The record headings below are this
view's SEED (FORMAT.md "Body as seed"): on first load the runtime
scaffolds vault/ with one id'd note file per record, so the view
arrives pre-loaded; this prose stays here, it is not data.
