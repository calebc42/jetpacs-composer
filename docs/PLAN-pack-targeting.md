## Stages (evening-sized, individually landable; order below is the dependency spine)

### S4.0 — Unknown-source preservation (the version-gate-brick guard; do FIRST)
Both parsers must survive a document whose `:SOURCE:`/`:ACTIONS:` reference vocabulary
they don't know — preserve on round-trip, degrade the view, never reject the whole app.
elisp `jetpacs-crud-orgapp.el` + `OrgCodec.kt`, plus shared `parser-parity.manifest` rows
(that manifest is the cross-parser contract — extend IT, both suites consume it).
Accept: round-trip fixtures with an unknown `pack:` source survive byte-faithfully on
both sides; suite + `./gradlew test` green.

### S4.1 — Pack manifest in the composer (Kotlin)
`model/PackManifest.kt`: parse `glasspane-pack.json` (pack_id/version/min_jetpacs_api/
feature/depends/sources/actions; closed field types mirror
`jetpacs-source-field-types`). Installed-manifest store + selection (a local directory of
`*-pack.json` files the user points at; "selected manifest" is per-app state).
Source/action pickers driven by the **vendored contract.json + the selected manifest**
(replace the free-text field in `ViewForm.kt`; ActionEditor offers the manifest's
annotated actions with their typed args). `node_types`→fallback awareness folded into
`ModelOps.validate` as **warnings only** (per-connection, permissive when absent).
Accept: manifest parse round-trip test against the real committed glasspane-pack.json
(stage it as a fixture); picker state tests; validate-warning tests.

### S4.2 — FORMAT 4 (pack references become emittable)
`#+JETPACS_PACK:` file-level declaration (pack id + minimum pack version), parsed both
sides; **emit-gating**: writer emits 3 when no pack feature is used, 4 when any
`pack:` source/action or `#+JETPACS_PACK:` is present; accept ≤ 4; reject > 4 (both
parsers — mirror the elisp `>` gate). FORMAT.md rewrite of the affected sections
(DEPENDS vs PACK: raw slugs stay `#+JETPACS_DEPENDS:`; manifest-backed packs are
`#+JETPACS_PACK:`). Parity manifest rows for: format-4 pack doc accepted, format-5
rejected, pack doc emitted at 4, non-pack doc still at 3.
Accept: parity + goldens (regen deliberately, review the diff); OrgCodecTest format-gate
test extended.

### S4.3 — Runtime binding (elisp; the fail-closed contract)
- `pack:` **sources**: the runtime resolves `pack:<id>/<name>` by requiring the selected
  manifest's `feature` (generated *trusted* bundle code may `require` that fixed feature —
  app data can NEVER choose a feature, download code, or trigger installs; SPEC §5), then
  `jetpacs-source-query` on `<name>`. Missing feature / unregistered source / manifest
  mismatch ⇒ **fail closed**: unavailable-view diagnostic body (the "needs vulpea"
  degradation pattern at `jetpacs-crud.el` is the shape), **no mutation dispatch**.
- `crud.pack.action` **server-side handler**: dispatch ONLY to action names present in
  the selected manifest's action list ∩ `jetpacs-action-handlers` (closed vocabulary,
  never arbitrary funcall); typed args pass through; unknown/mismatched token ⇒ clean
  `user-error`, no dispatch.
- Duplicate pack ids across installed manifests ⇒ fail closed at selection time.
Accept: ERT — a fake pack (test-local defsource/defaction + a toy manifest) round-trips
render + action; missing-feature and unknown-action paths produce the diagnostic and
provably dispatch nothing; bundle smoke stays green on bare core.

### S4.4 — Manifest-driven dependency bootstrap
`Deployer.kt`: the install list comes from the selected manifest's `depends` (falling
back to `requiredDepends()`'s vulpea/org-ql for non-pack apps); keep the
condition-case/offline-retry/idempotent shape and the autosync setup. Non-elisp deps
(the engines plan flags rg/fd-class binaries) are surfaced as "not installable from
MELPA — install via Termux" warnings, never attempted.
Accept: snippet/bootstrap unit tests over a manifest fixture; FORMAT.md device-setup
section updated.

  file is added.
- `parser-parity.manifest` is the cross-parser contract; the elisp parser is the oracle.
- jetpacs core stays untouched (no pack registry). If a core seam is genuinely missing,
  promote it explicitly (API-STABILITY entry + api minor bump + contract.json regen) — do
  not poke internals.
- Addressing/mutation invariants to preserve (master list): positional COLTYPES/SCHEMA,
  save→reindex→repush, closed `crud.*` vocabulary + server-side resolve, logical
  addressing (`(line,col)` cells / `(idx,text)` checklist / `:ID:` records).
